# Atividade: A Arquitetura da Resiliência — Tratamento de Erros e Avaliação de Soluções

**Disciplina:** Pensamento Computacional
**Professora:** Kadidja Valéria
**Projeto base:** Sistema de organização de dados da turma (Lista, Grafo e Hierarquia)

## Objetivo

Aplicar o **tratamento de erros** diretamente sobre o projeto desenvolvido na
atividade "Do caos à organização" (pasta `Organizacao_de_Dados/`). O projeto
de origem foi expandido com um pequeno script (`sistema_turma.py`, descrito nos
arquivos abaixo) que processa a lista de alunos e suas notas — e que continha
erros propositais de **sintaxe**, **lógica** e **execução**.

## Etapas realizadas

1. **Identificar** — localização dos erros ocultos no algoritmo original
   (ver `Algoritmo_Erros.md`).
2. **Corrigir** — refatoração do código com justificativa técnica para cada
   alteração (ver `Algoritmo_Corrigido.md`).
3. **Avaliar** — reflexão crítica sobre clareza, eficiência e escalabilidade
   da solução final (ver `Avaliacao.md`).

## Conteúdo desta pasta

- `README.md` — este arquivo.
- `Algoritmo_Erros.md` — versão original do algoritmo, com mapeamento dos bugs encontrados.
- `Algoritmo_Corrigido.md` — código corrigido, comentado, com as justificativas da refatoração.
- `Avaliacao.md` — reflexão sobre eficiência, clareza e escalabilidade da solução final.

## Fundamentação

Conforme a aula "A Arquitetura da Resiliência": um sistema inteligente é,
acima de tudo, um sistema **preparado para errar** — o objetivo não é apenas
encontrar *se* existe erro, mas *qual* é o erro e *onde* ele está.
